# 模型下载、部署、校验 — 完整方案

## 一、下载层（确保所有模型可下载）

### 1.1 补全 Python 下载注册表
**文件**: `workers/download_models.py`

在 `MODELS` dict 中新增缺失条目（对应 DB seed 中已有的模型）：

| 新增 key | 类型 | 来源 |
|----------|------|------|
| `chattts` | tts | HF: `2noise/ChatTTS` |
| `piper` | tts | HF: `rhasspy/piper-voices` (onnx 格式) |
| `bge-small` | embedding | HF+MS: `BAAI/bge-small-zh-v1.5` |
| `jina-v3` | embedding | HF: `jinaai/jina-embeddings-v3` |

更新 `--list` 命令输出格式，增加类型、大小信息。

### 1.2 删除无下载支持的 DB 种子
**文件**: `src-tauri/src/storage/schema.sql`

移除 `model_chattts`、`model_piper`、`model_bge_small`、`model_jina_v3` 的种子 INSERT（这些模型的 provider 在 Python 端没有实现，从 DB 删掉避免用户点击下载后报"未知模型"）。

> 备选方案：保留种子但标记为 `status='unavailable'`，前端显示"暂不支持"。

## 二、部署层（确保模型下载后能被 Worker 找到）

### 2.1 修复 CosyVoice 路径
**文件**: `workers/tts_worker.py`

当前硬编码路径 `"pretrained_models/CosyVoice2-0.5B"` 与下载目录 `data/models/cosyvoice` 不匹配。

**修复**: 增加环境变量 `COSYVOICE_MODEL_PATH` 或命令行参数，默认指向下载目录。在 `_get_cosyvoice()` 中使用该路径。

### 2.2 ASR/Embedding 模型路径适配
**文件**: `workers/asr_worker.py`

faster-whisper 和 sentence-transformers 通过 huggingface_hub 自动下载到 `~/.cache/huggingface/`，手动下载到 `data/models/whisper` 不被使用。

**修复**: 增加 `model_path` 参数支持本地模型目录：
- ASR: `WhisperModel(model_size_or_path, ...)` faster-whisper 原生支持本地路径参数
- Embedding: `SentenceTransformer(model_name_or_path)` 同样支持本地路径

### 2.3 模型下载目录统一
**文件**: `src-tauri/src/commands/model.rs`

下载脚本默认输出到 `--output {modelsDir}`，Rust 校验路径为 `{modelsDir}/{model_id}`。两者一致 → 无需修改。

## 三、校验层（确保模型下载后可用）

### 3.1 下载后自动校验
**文件**: `src-tauri/src/commands/model.rs` + `workers/download_models.py`

在 Python 下载成功后增加 `LOADCHECK:<model>` 步骤：
- ASR/whisper: 尝试 `WhisperModel(...)` 加载 → 报告 `LOADCHECK:OK` 或 `LOADCHECK:FAIL`
- TTS/pyttsx3: 尝试 `pyttsx3.init()` → 返回引擎状态
- Embedding: 尝试 `SentenceTransformer(...)` 加载

Rust 端解析 `LOADCHECK:` 行，成功后 DB status 设为 `"downloaded"`，失败则设为 `"failed"` 并记录错误信息。

### 3.2 完善 `test_model` 命令
**文件**: `src-tauri/src/commands/model.rs`

| 类型 | 当前行为 | 修复 |
|------|---------|------|
| Embedding | 返回"不支持测试" | 发送 ping 到 embedding worker |
| LLM(本地) | 永远返回成功 | 尝试加载 GGUF 文件验证 |
| ASR/TTS | ping worker | 增加超时重试 + 详细错误信息 |

### 3.3 前端状态卡片改进
**文件**: `src/components/ModelDownloadCard.vue`

- ✅ 下载中：从 store 的 `downloadTasks` 读取实时进度百分比
- ✅ 已下载但未启用：显示"测试"按钮（不要求先启用）
- ✅ 下载失败：显示红色"重试"按钮 + 错误信息 tooltip
- ✅ 增加"校验"操作（调用 test_model），显示成功/失败结果

### 3.4 启动时健康检查
**文件**: `src-tauri/src/lib.rs` + `frontend`

在应用启动完成后（延迟 2 秒）执行：
1. 检测 Python 是否可用
2. 检测 Ollama 是否可用
3. 将结果通过 Tauri event `app-health` 推送到前端
4. 前端在 `AppLayout.vue` 监听并显示通知栏状态
